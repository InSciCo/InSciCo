Create Simple Stack

In this step we will use LazyStack to create and test a very simple serverless stack. We present these steps assuming you are using the Visual Studio IDE (Windows). If you are using Visual Studio IDE (MacOS) or the .NET CLI, substitute those commands as you saw illustrated in the associated tutorial.

Hello World Stack

In this example we will be creating the proverbial "hello world" stack. The stack will use HttpApiGateway and a single Lambda project. A single Http GET method "SayHello" will return a string result "Hello from MyStack", where MyStack is the name of the stack you publish.

1.1 Create MyStack Solution

Pick a unique name for your stack and use that instead of "MyStack" throughout this tutorial so your work doesn't conflict with another Tutorial user. Stack names should be short, alphanumeric and start with a character. To guarantee uniqueness in your organization, you may want to name the stack after your AWS Account Name; ex: OrgDevJoe.

  1. Create a LazyStackTutorial Folder (you may already have this folder if you have performed another LazyStack Tutorial).
  2. Launch Visual Studio.
  3. Click on Create a new project. The "Create a new project" dialog appears.
  4. Type "LazyStackWebApi" into the search field. A list of project templates is presented.
  5. Double Click on the LazyStackWebApi project template. The "Configure you new project" dialog appears.
  6. Enter the stack name as the Project Name.
  7. Edit the Location to point to the LazyStackTutorial folder.
  8. Click on the Create button. A solution for your stack project is created.
  9. Select Visual Studio menu item "Build → Rebuild Solution". The solution compiles successfully.

1.2 Download SimpleStack.yaml and run LazyStack Generator

  1. Click to Download SimpleStack.yaml file
  2. Move the SimpleStack.yaml from your Downloads folder to the Solution's root folder.
  3. Rename "SimpleStack.yaml" to MyStack.yaml where MyStack is your stack name.
  4. Select Visual Studio menu item "Tools/LazyStack - Generate Projects". Multiple projects and solution folders are created.
    • Controllers folder
      • HelloController project
    • Lambdas folder
      • Hello project
    • "Solution Items" folder (files are in solution's root folder)
      • LazyStack.yaml
      • LocalApis.json
      • MyStack.yaml (your stack name)
      • SAM_Review.yaml
    • Stacks folder
      • Dev folder
        • serverless.template
    • MyStackClientSDK project - with your stack name
    • MyStackSchema project - with your stack name

1.3 Implement the MyStack application Controller

In this step we implement a simple controller.

  1. Create MyStackRepo project from "C# Class Library (.NET Core)" template.
    1. Right Click on Solution MyStack item in the solution explorer. The context menu appears.
    2. Select the Add → New Project context menu item. The "Add a new project" dialog appears.
    3. Find the "C# Class Library (.NET Core)" project template.
    4. Double click on the "C# Class Library (.NET Core)" template. The Configure your new project window appears.
    5. Enter MyStackRepo as the project name - using your stack name.
    6. Click the Create button. The project is created with a default class Class1.cs.
  2. Edit the Class1.cs file.
  3. Build the project.
