SMF Step 2 - Create System

Prerequisites:
- Create AWS Organization complete
- Unique email for root user of new System Test AWS Account
- Unique email for root user of new System Production AWS Account
- SysCode (system short code to use for naming accounts that are part of system)
Estimated Time: 4 minutes
Estimated Cost: none
Perform this step on the Management Workstation.

Systems are comprised of a System Test Account and a System Production Account. Systems contain one or more Serverless Stacks, with each stack associated with a GitHub repository.

  • Step 2.1 - Create System QuickStart
  • Step 2.2 - Create Test Account
  • Step 2.3 - Create Production Account
QuickStart

Performing "2.1 - Create System QuickStart" performs the same processing as steps 2.2 and 2.3 and Config Repository steps 3.1, 3.2 and 3.3. Unless you have special requirements you should use QuickStart to create a System. If you require more than one Serverless Stack in a System, you can use the "Config Repository" steps to add additional Serverless Stack repositories to an existing stack created with QuickStart.

We recommend that you use "2.1 Create System QuickStart" to create an initial System called Tut (short for Tutorial) to be used by you and your team to perform the LazyStackSMF tutorial.
