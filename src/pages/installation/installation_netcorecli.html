.NET CLI Tool

The LazyStack generator is available as a .NET CLI Tool. Use this tool if you are not using Visual Studio (Windows) for development or need to automate deployment tasks.

Prerequisites

  • AWS Account
  • AWS CLI
  • AWS SAM CLI
  • .NET 5.0

To install the LazyStack CLI Tool, open a console and issue this command:

$ netcore tool install lazystackapp -g

To update to the latest LazyStack generator, open a console and issue this command

$ netcore tool update lazystackapp -g

LazyStack Tool Synopsis

Generate Projects
lazystack projects [<solutionfilepath>]

If this command is executed in the solution folder then the solutionfile path is not required as the .sln file will be used.

This command ingests your application's OpenAPI specification to generate your C# projects and Serverless Application Model (SAM) file(s). In addition, LazyStack reads generator "directives" from your application's LazyStack.yaml file to control various generation options.

Note that your applications OpenAPI specification must have the same name as the solution file but have a .yaml suffix.

Generate AwsSettings file
lazystack settings <stackname> [<filepath>] [-n <profilename>]

This command uses AWS Cloud Formation to retrieve settings from the specified <stackname> and then write these settings out to "AwsSettings.json" or to the specified <filepath>. Your "default" AWS profile will be used unless you specify a different profile with the -n option.

Quick Start

  1. Create AWS Account
  2. Install AWS CLI
  3. Install AWS SAM CLI
  4. Install .NET 5.0
  5. Open a command window and install the .NET LazyStack Tool 
      • $ netcore tool install lazystackapp -g
        Projects and configuration files are generated and added to solution.
  6. Create a new project using the LazyStackWebApi Template.
  7. Author an OpenApi specification for your application stack and place it in the solution folder.
  8. Open a Command Window and cd to the solution folder and use the .NET CLI Tool to generate projects.
    • $ lazystack projects
      Projects and configuration files are generated and added to solution.
  9. Update LazyStack Directives in the LazyStack.yaml configuration file in the solution folder.
  10. Implement service-side application code for the the generated Controller Interfaces.
  11. Create a client application that references the generated application ClientSDK and uses the LazyStackAuth Nuget package.
  12. Using AWS Dashboard or AWS CLI, create an s3 bucket for deployments.
  13. Open a Command Window and cd to the stack folder you want to deploy. Example <solutionfolder>\Stacks\Dev 
    • $ sam deploy -t serverless.template --stackname <stackname> --s3-bucket <s3bucket>
      • Your stack is published. See here for sam deploy documentation
  14. Open a Command Window and cd to the stack folder you want to generate an AwsSettings.json file for. Example <solutionfolder>\Stacks\Dev 
    • $ lazystack settings <stackname>
    An AwsSettings file will be generated.
  15. Update client application to use the AwsSettings file as an embedded resource.
