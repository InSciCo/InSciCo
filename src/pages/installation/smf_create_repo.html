SMF 3.1 Create GitHub Repository

Prerequisites: Create GitHub Account (1.4) complete
Estimated Time: 2 minutes
Estimated Cost: none

See this newtab}}> Creating a new repository documentation on GitHub.

Note: The new repository must be accessible from the "Personal account" used to generate the "Personal Access Token" in "Create GitHub Account" (step 1.4). We suggest this repository be part of a GitHub Organization.

  1. Log in to GitHub.
  2. Create the repository if you don't already have one for this stack.
    1. Switch to the GitHub Organization. This done using the drop-down combo in the upper left corner of the GitHub dashboard.
    2. Click on the "New" button, next to the Repositories heading, to create a new repository. The "Create a new repository" page appears.
    3. Enter a Repository name. If this is a repository for a developer who will be performing the SMF Tutorial, we suggest you use the developer's account name with PetStore appended. ex: OrgDevJoePetStore.
    4. Enter a repository description. Example: "Joe's SMF Tutorial Repository"
    5. Click on the "Private" radio button to make the repository private.
    6. Check the "Add a README file"
    7. Check the "Add a .gitignore file" checkbox. The .gitignore template drop-down menu appears.
    8. Select "VisualStudio" from the .gitignore template drop-down menu.
    9. Check the "Choose a license" checkbox. The License drop-down menu appears.
    10. Select "MIT License" from the License drop-down menu.
    11. Click the "Create repository" button.
  3. Navigate to the Code tab for the repository.
  4. Click on the "Code" button. The download options drop-down menu appears.
  5. Under the HTTPS option you will see the url for your repository.
  6. Click the clipboard option next to the URL to copy that URL.
  7. Make a note of this URL as you will need it in the following steps.
