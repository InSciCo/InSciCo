SMF 1.4 Create GitHub Account

Prerequisites: AWS Mgmt. Account Setup Complete
Estimated Time: 2 minutes
Estimated Cost: none
Perform this step on the Management Workstation.

A GitHub account is required by LazyStack SMF. You will be creating one GitHub repository for each serverless stack in your AWS Organization. See the User Guide newtab}}>GitHub Integration page for more details.

GitHub allows you create Organization accounts. You log into GitHub using a personal account and that account is a member of an Organization. SMF Setup requires you have a GitHub personal account that has full access to the repositories your team will be using to develop serverless stacks. Later on, we use GitHub Personal Access Tokens to configure AWS CodeBuild access to the GitHub repositories. So, in this step, we generate and save a "Personal access token" we will use later on.

  1. Visit newtab}}>Join GitHub to create a GitHub Account if you don't already have on. If you don't already have one, create a GitHub Organization so you can associate GitHub developer accounts with repositories created under that Organization.
  2. Generate a Personal Access token:
    1. Log into the GitHub personal account that has access to the repositories you want to use.
    2. Click on the user profile icon in the upper right of the page. The user profile menu appears.
    3. Select the Settings menu item. The "personal account page" appears.
    4. Select "Developer Settings" from the left hand menu. The "Developer Settings" page appears.
    5. Select "Personal access tokens" from the left hand menu. The "Personal access tokens" page appears.
    6. Click on the Generate new token button. The "Confirm access" page appears. Note: GitHub may request your password at this point, provide it if requested. The "New personal access token" page appears"
    7. Enter a descriptive note, ex: "AWS CodeBuild Access".
    8. Check the "repo" option.
    9. Check the "admin:repo_hook" option.
    10. Click the "Generate token" button. The generated token is displayed.
    11. Copy this token and save it in a file named "GitHubToken.txt" in the OrgMgmt folder. Replace "Org" with your OrgCode.
  3. Sign out of the GitHub account.
