Configure LazyStack Software Management Framework (SMF)

The LazyStack SMF provides an end-to-end Software Management Framework integrating AWS Services, GitHub and the LazyStack developer tools. Take a close look at the diagram below, we will be following the SMF Setup Steps 1 through 3 to create your first AWS Organization, with one system (called Tut). Later, in the LazyStack SMF Tutorial, you use step 4 to add serverless application stacks to the Tut system. After completing the tutorial you add additional Systems and Stacks when you need them. Keep in mind that AWS has a default limit of 4 accounts per organization, you can ask them to increase this limit as necessary.

Configuring your AWS Organization

Follow these steps to prepare the SMF for your team to start doing development.

  • Step 1 - Create AWS Organization
    • 1.1 Configure Management Workstation
    • 1.2 Create AWS Management Account
    • 1.3 Create AWS Organization
    • 1.4 Create GitHub Account(s) - one may be all you need but multiple GitHub Accounts may be used

  • Step 2 - Add Developer(s)
    • 2.1 Create Developer AWS Account (each developer has their own "sandbox" account)
    • 2.2 Configure Developer's Workstation

  • Step 3 - Create System(s)
    • We define a System as a collection of AWS Resources managed together (serverless stacks are AWS Resources). Systems are used to segregate unrelated applications; for example different line-of-business, product or customer. System's are managed in two AWS Accounts: Test and Production.
    • 3.1 Create AWS Test Account for System
    • 3.2 Create AWS Production Account for System

  • Step 4 - Create Stack(s)
    • Serverless Stacks are a collection of resources deployed together as a unit. Each Serverless Stack is associated with a single GitHub Repository. A System may contain one or more Serverless Stacks.
    • 4.1 Create GitHub Repository
    • 4.2 Configure Continuous Integration and Continuous Deployment for System Test Account
    • 4.3 Configure Continuous Deployment for System Production Account

For a comprehensive overview of the deployment, visit the Deployment Summary.
