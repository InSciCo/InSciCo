AWS Management Account Setup

We recommend you create a new AWS Account if you intend to use our guides to create an AWS Organization.

Account Names

We recommend that you use a short "organization code" (OrgCode) to prefix all accounts and users in your AWS Organization. For instance, we recommend that you name your AWS Management Account something like "OrgMgmt" - where "Org" is the OrgCode.

We recommend the OrgCode be no longer that 2 or 3 characters. It must be alpha-numeric and have no spaces. This short OrgCode will be very helpful in keeping track of accounts belonging to different organizations should you end up managing or developing in more than one organization on the same Workstation.

If you don't have a formal organization to derive the OrgCode from, you may want to consider using this prefix to describe an App Stack you are building.

Replace "Org" with your OrgCode whenever you see it in the guide steps.

New AWS Management Account Setup

Prerequisites: A valid email address (or alias) not used on any other AWS Account.
Estimated Time: 10 minutes, Costs: none
Perform this step on the Management Workstation. The Management Workstation can also be a Development Workstation.

  1. newtab}}>Sign up for an AWS account using your email address - this will become your AWS "Management" account. Name this account "OrgMgmt" - replacing "Org" with your OrgCode.

    Make a note of the email address and password used to create the account. You will need this to login as the Account's "root" user.

    Important: If you already have an AWS Account associated with your email, you will need to create an email alias to create another AWS account. You can't use the same email address with more than one AWS Account.

    Tip: If you are using a Gmail account you can create and use a dynamic alias using the special '+' operator. If your Gmail address is "me@gmail.com", you can use the email address alias "me+OrgMgmt@gmail.com". Any email sent to me+OrgMgmt@gmail.com will show up in your regular Gmail inbox.

  2. Sign out. We want to ensure we can sign in.

  3. newtab}}>Sign into your AWS Management Console as Root user. The AWS Management Console appears.

  4. Create a "OrgMgmt" OS folder. Replace "Org" with your OrgCode. This folder will be used to store account information and scripts for managing the account and AWS organization.

  5. Create OrgMgmt IAM User and generate a OrgMgmt_credentials.csv file
    Replace "Org" with your OrgCode in the following steps.
    1. In the AWS Services section, find and select "IAM". The IAM dashboard appears.
    2. In left-hand navigation bar, select Users. The user list appears. As this is a new Account, there are no IAM users.
    3. Click the Add user button. The Add user (step 1) page appears.
    4. Enter "OrgMgmt" in the User name field. Replace "Org" with your OrgCode.
    5. Check the Programmatic access checkbox.
    6. Check the AWS Management Console access checkbox. The Console password and Require password reset options appear.
    7. Select the Custom password radio button. Enter a custom password.
    8. Uncheck the Require password reset checkbox.
    9. Click the Next:Permissions button. The Add user (step 2) page appears.
    10. Click the Create group button. The Create group dialog appears.
    11. Enter "Administrators" in the Group name field
    12. Click the checkbox for the Policy named AdministratorAccess.
    13. Click the Create group button. The dialog closes.
    14. Click the Next: Tags button. The Add user (step 3) page appears.
    15. Click the Next: Review button. The Add user (step 4) page appears.
    16. Click the Create user button. The Add user (step 5) page appears.
    17. Click the Download .csv button.
    18. Rename the "new_user_credentials.csv" file to "OrgMgmt_credentials.csv" and move it to the OrgMgmt folder. Replace "Org" with your OrgCode.
    19. Click the Close button. The IAM console appears.
  6. Sign out of your AWS Management Account.

  7. newtab}}> Install AWS CLI

  8. Create OrgMgmt AWS CLI Profile
    1. Open a Command Prompt / Terminal
    2. Issue the following command: (replacing "Org" with your OrgCode) 
      aws configure import --csv file://OrgMgmt_credentials.csv
      This creates the OrgMgmt profile.
    3. Issue the following command: (replacing "Org" with your OrgCode, and editing the region if necessary) 
      aws configure set region us-east-1 --profile OrgMgmt
    4. Issue the following command: (replacing "Org" with your OrgCoe)": 
      aws configure list --profile OrgMgmt
      Example Output: 
      
       Name                    Value             Type    Location
       ----                    -----             ----    --------
    profile                   OrgMgmt          manual    --profile
access_key     ****************KZ6S shared-credentials-file
secret_key     ****************8b3b shared-credentials-file
    region                us-east-1      config-file    ~/.aws/config

  9. Secure your Root User Account Credentials.
    You will not be using the Root User credentials often. Store them securely where you can find them when you need them. Never share these credentials with third-parties like vendors or consultants. Somebody with your Root User security credentials can do anything in your account.
  10. newtab}}>Install AWS SAM CLI
Step Summary

In this step we created a new AWS Management Account, added an IAM User "OrgMgmt" with AdministratorAccess, Installed the AWS CLI on our workstation and configured a AWS CLI profile for the the IAM User "OrgMgmt". Now we can perform most administrative tasks on the Management Account using the AWS CLI.

You can now proceed to run the LazyStack tutorial in the Management Account for POC or educational purposes or continue with setting up the AWS Organization SDLC.
