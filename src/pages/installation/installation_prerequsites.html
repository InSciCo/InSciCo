Prerequisites

Before you start the installation process, please review these prerequisites.

Management Workstation

This is the computer from which you will be creating/managing your AWS Organization and Accounts. You must have administrative rights on this computer to install and use a variety of software.

Development Workstation

This is the computer a developer uses to develop the application code that runs in the AWS Stack. Each developer will have their own AWS Account. You may use the Management Workstation as a Development Workstation. You must have administrative rights on this computer to install and use a variety of software.

Pick an OrgCode

Throughout the setup process we use an organization prefix, the OrgCode, in AWS Account names and AWS organizational Unit names. For instance, we name the AWS Management account "OrgMgmt". You need to pick your own "OrgCode" to use for your organization or app and wherever you see "OrgMgmt", replace the "Org" with your short OrgCode prefix.

We recommend the OrgCode be no longer that 2 or 3 characters. It must be alpha-numeric and have no spaces. This short OrgCode will be very helpful in keeping track of accounts belonging to different organizations should you end up managing or developing in more than one organization on the same Workstation
Here is an example of the naming with an OrdCode of "Az":

AWS Item Name in Docs Name with OrgCode "Az"
Management Account OrgMgmt AzMgmt
Dev Organizational Unit OrgDevOU AzDevOU
Dev Account OrgDevJoe AzDevJoe
Test Organizational Unit OrgTestOU AzTestOU
Test Account OrgTest AzTest
Prod Organizational Unit OrgProdOU AzProdOU
Prod Account OrgProd AzProd

* Note: There many be more than one development account.

Email Account & Alias

Each Amazon account has a Root User whose credentials include an email address and password. The email address must be unique across all accounts on AWS; e.g. you may not use the same email address for more than a single AWS Account. You will create 4 accounts when setting up your AWS Organization, so you will need 4 email addresses (or aliases).
If you are using Gmail, you can use the Gmail dynamic aliasing technique as follows:

MyEmail@gmail.com -- existing email address

  1. MyEmail+OrgMgmt@gmail.com -- root user for management account
  2. MyEmail+OrgJoeDev@gmail.com -- root user for Joe's development account
  3. MyEmail+OrgTest@gmail.com -- root user for test account
  4. MyEmail+OrgProd@gmail.com -- root user for production account

The "+" character makes the email address a dynamic alias.
If you are not using Gmail, create email addresses or aliases in your email software instead.

As mentioned above, you would replace the "OrgCode" in the example email addresses with your selected OrgCode; if that was "Az" then your email addresses would be:

  1. MyEmail+AzMgmt@gmail.com -- root user for management account
  2. MyEmail+AzJoeDev@gmail.com -- root user for Joe's development account
  3. MyEmail+AzTest@gmail.com -- root user for test account
  4. MyEmail+AzProd@gmail.com -- root user for production account
