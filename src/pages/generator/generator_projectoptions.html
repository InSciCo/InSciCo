ProjectOptions Directive

This directive allows you to add/override options for project files and code generated in LazyStack projects. These options include:

  • PackageReferences - specify the package references to insert into a project file.
  • ProjectReferences - specify the project references to insert into a project file.
  • ServiceRegistrations - specify class DI registrations for a project.
  • Properties - specify properties to add to the project file.
  • CodeUriTarget - specify what build target to upload to AWS.
  • Runtime - specify what AWS Lambda Runtime to use.

Many project options have default values:

# LazyStack Version 1.0.0
ProjectOptions:
  ReferenceApiProjects:
    PackageReferences:
      Microsoft.AspNetCore.Mvc.NewtonsoftJson: "3.0.0"
      Microsoft.Extensions.Configuration.Json: "3.1.0"
      Swashbuckle.AspNetCore: "5.6.2"
  
  ClientSDKProjects:
    PackageReferences:
      AwsSignatureVersion4: "1.3.1"
      System.ComponentModel.Annotations: "4.5.0"
      JsonSubTypes: "1.5.2"
      Newtonsoft.Json: "12.0.1"
  
  LambdaProjects: 
    CodeUriTarget: "bin/Release/netcoreapp3.1"
    Runtime: dotnetcore3.1
    PackageReferences:
      AWSSDK.S3: "3.3.110.58"
      AWSSDK.Extensions.NETCore.Setup: "3.3.100.1"
      Amazon.Lambda.AspNetCoreServer: "5.1.0"
