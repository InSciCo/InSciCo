Deployment Checklist

When you build stacks, you should use the following checklist to ensure you have addressed security, performance and cost concerns. If you are releasing the stack for others to deploy, we recommend you document the deployment checklist in a deployment guide.

Note: Items with an ♦ are not usually required in a pure serverless stack.

  • Deployment Guide Documentation
    • Introduction
      • Describe Use-Cases for software
      • Deployment Resources
      • Deployment Options
    • Prerequisites and Requirements
      • Technical Prerequisites
      • Specialized Skills or Knowledge Required
      • Environment Configurations
    • Architecture Diagrams
      • AWS services and relationships among them.
      • Customer Data storage is labeled
      • Diagrams use AWS Simple Icons
      • ♦ Networking:
        • ♦ VPCs
        • ♦ subnets
        • ♦ security groups
        • ♦ NACLs
        • ♦ Ingress/egress mappings
      • Integration Points
        • Third-party assets/APIs
        • ♦ On-premises/hybrid assets
    • Security
      • Links to IAM and IAM best practices documentation
      • Attestation that stack does not require root privileges to deployment or operation
      • Attestation that stack follows least privilege for access granted as part of deployment
      • List any public resources with bucket policies allowing public access
      • List and describe any IAM role and policy user is instructed to create
      • List and describe the purpose and location of any key the user is instructed to create
      • Provide clear instructions on maintaining any stored secretes stored in AWS Secrets Manager
      • Provide details on where customer sensitive data is stored
      • Provide details on data encryption configuration used in data stores
      • Provide links to any third-party stacks that need to be deployed with this stack
    • Costs
      • List billable services included in stack and indicate if the service is mandatory/optional
      • Provide operational cost and licensing cost
    • Sizing
      • ♦ Guidance for EC2 Instance type and size selection
      • ♦ Guidance for EBS volume type and size selection
      • ♦ Guidance on instance size selection for managed AWS Services
      • Guidance on AWS DynamoDB read and write capacity unit estimates
    • Deployment Assets
      • Setting up automated backups
      • Restoring data from a backup
      • Step-by-step instruction for recovering from service failure
      • Step-by-step instructions for recovering from AZ failure
      • Managing service limits for Disaster Recovery
      • Document Recovery Time Objective (RTO)
      • Document Recovery Point Objective (RPO)
    • Routine Maintenance
      • Step-by-step guide for rotating programmatic system credentials and cryptographic keys
      • Prescriptive guidance for software patches and upgrades
      • Prescriptive guidance for managing licenses
      • Prescriptive guidance for managing AWS Service Limits
    • Emergency Maintenance
      • Step-by-step instructions on handling fault conditions
      • Step-by-step instructions on recovering the software
    • Support
      • How to receive support
      • Technical Support Tiers
      • Service Level Agreement by Tier
