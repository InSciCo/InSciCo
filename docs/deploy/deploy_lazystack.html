LazyStack Deployment Technical Base Requirements

LazyStack is a tool that generates AWS SAM Templates that are used to publish AWS Stacks. In addition to the AWS Account Basics, the following additional requirements apply to LazyStack.

Technical Skills

  • C# and Command Shell
  • .NET
  • OpenApi Specification

AWS Command Line Interface

"The AWS Command Line Interface (AWS CLI) is a unified tool that provides a consistent interface for interacting with all parts of AWS. AWS CLI commands for different services are covered in the accompanying user guide, including descriptions, syntax, and usage examples."
AWS Command Line Interface

AWS Serverless Applications Model (SAM)

The AWS Serverless Application Model (SAM) is an open-source framework for building serverless applications.
AWS SAM

AWS SAM Command Line Interface

"The AWS SAM CLI is a command line tool that operates on an AWS SAM template and application code. With the AWS SAM CLI, you can invoke Lambda functions locally, create a deployment package for your serverless application, deploy your serverless application to the AWS Cloud, and so on."
AWS SAM CLI

AWS ToolKit for Visual Studio (Windows Only)

"The AWS Toolkit for Visual Studio is a plugin for the Visual Studio IDE that makes it easier for you to develop, debug, and deploy .NET applications that use Amazon Web Services. The Toolkit for Visual Studio is supported for Visual Studio versions 2013 and later."
AWS Toolkit for Visual Studio

Source Code Management

A source code management system like GitHub. Many other Git based services are also excellent choices. Some of these include BitBucket, GitLab and AWS CodeCommit. We recommend that you implement the Git Flow workflow, Git Flow is supported by most Git repository services.

Continuous Integration / Continuous Deployment (CI/CD) System

It is highly recommended that you use a CI/CD system. We are currently working on an example workflow based on Git Flow, Pull Requests and the use of the AWS CodePipeline service. LazyStack easily integrates with CI/CD pipelines.
