LazyStack Executive Overview

LazyStack's mission is to enable technical entrepreneurs to quickly field mobile and web applications leveraging scalable cloud-based resources at the lowest initial cost, without sacrificing security, scalability or features. LazyStack provides all the necessary setup instruction and tools necessary develop, test, maintain and deploy AWS Serverless Application Model (SAM) "application stacks".

LazyStack Software Management Framework (SMF)

Managing software development can be hard. Developing the tools, practices and documentation necessary to effectively manage software development can be even harder. The LazyStack SMF includes the tools, templates, scripts, practices, and examples to set up an effective Software Management Framework with AWS Serverless Stacks in mind. Use our "suggested" SMF to get started quickly with effective best-practices or just review our SMF for hints on how to integrate the LazyStack development tools into your existing Software Management Practices.

LazyStack SMF includes:

  • Secure AWS Organization and Accounts following all AWS best practices
  • Integrated Software Source Management using GitHub, integrated with AWS and development tools
  • Team coordination using the popular Feature Branch Workflow with GitHub Pull Requests (Pull Requests are requests by developers for approval of new features)
  • Application quality assurance with Continuous Integration (CI)
  • Timely production updates with Continuous Deployment (CD)
  • Rapid development/prototyping leveraging individual AWS Developer accounts/sandboxes

LazyStack Model Driven Development (MDD) Tools

LazyStack MDD brings three great technologies together to delivery apps at the lowest cost and fastest time-to-market.

Model Driven Development generates code and resource configurations from a "specification" document. The code can be regenerated when the specification document is updated or the code generation "pattern" is improved. This increases the responsiveness of the development process to changing requirements in features or underlying technologies.

Model Driven Development reduces costs for initial code development and ongoing code maintenance. Web and mobile clients communicate with Internet services that publish a Application Program Interface (API). These APIs define each available operation and the types of data that may be exchanged. A common "language" for defining an service API is the OpenAPI specification.

Reading an OpenAPI specification is not difficult so they are an ideal tool for business stakeholders and the development team to collaborate on application features dependant on the client calling the service. It is often useful to enumerate the API operations an application feature will use. This improves the stakeholders understanding of how the application works and allows developers to better estimate and execute feature implementation.

LazyStack MDD tools ingest an OpenAPI specification and then generates all the plumbing code and AWS Resource specifications necessary to deploy and use AWS Serverless application stacks. This Model Driven Development (MDD) approach improves the management of your application requirements, and lowers development and DevOps costs. LazyStack reduces hand coding and manual AWS resource configuration so you see real business benefits:

  • Clear Requirements
    • An API specification reduces misunderstandings and wasted implementation cycles.
    • API specifications can be versioned, making it simple to manage incremental improvement in a product.
    • API specifications make it easy to partner with other parties.
    • API specifications make it easier to discuss application features with business stakeholders.
    • Tools exist to help author effective API specifications.

  • Lower Cost Development
    • Expensive hand-coding of "repetitive" plumbing code is eliminated.
    • Innovation is easier because new plumbing code and templates are automatically generated when the API specification changes.
    • Generated code is consistent, bug-free and reliably compliant with best-practices.
    • Developers are freed up to work on features that can't be generated.
    • Generated code is far less expensive to test and verify.
    • LazyStack publishes a number of open-source MIT licensed libraries to kick-start your app development.

  • Lower Cost DevOps
    • Leverages AWS Serverless Application Model - generates AWS SAM templates from API Specification.
    • Generated templates demonstrate correct configuration of AWS resources.
    • Generated templates are consistent, bug-free and reliably compliant to best-practices.
    • Generated templates can be easily extended with hand-coded resource specifications.
    • Manage Dev, Test and Prod stacks with AWS CodeBuild Continuous Integration and Continuous Deployment.

LazyStack makes innovation faster and less expensive by reducing the costs associated with code and stack configuration changes. LazyStack accelerates your time-to-market so you start monetizing your app earlier. Leveraging the AWS Serverless Application Model ensures your app has the potential to scale reliably and quickly and launch at low cost using AWS free-tier services.

Why did we build LazyStack?

Technical Entrepreneurs generally have more ideas than capital. LazyStack makes it easier to bring a fully functional application stack to market at a very low development and operational cost so more of these ideas can be market tested until viable monetization is demonstrable. The recent “Fail Fast” mantra does have some merit, but only if your application attempt doesn’t fail in the technical sense. LazyStack generates commercial grade production stacks that can be used in support of prototype apps but also for production apps.

A Technical Entrepreneur may be a single person or a team where the aggregate skill-sets include product definition, product development, product testing, product marketing/sales and product support. We leave general business management and capital raise out of this definition because it is often necessary to demonstrate an app with modest market acceptance before you can raise capital to scale up.

We would also observe that even in larger organizations it is sometimes a “skunk works” team that fits our definition of Technical Entrepreneur.

So, why did we usurp the term LazyStack? There’s an old adage, “Only lazy programmers write code generators and only code generators can produce bug-free code; thus only lazy programmers write bug-free code”. Tongue firmly in cheek here! Well, OK then, why not whole stacks?
