Lazystack for Visual Studio MacOS

Visual Studio for Mac supports extensions but submissions to the gallery are no longer being accepted by Microsoft so we don't provide a LazyStack extesnion for Mac. Instead, we use the Visual Studio for Mac External Tools capability to make it easy to call LazyStack from the IDE.

The AWS Toolkit is not avaiable for Mac either. However, AWS provides the SAM CLI that we can use instead. Just like LazyStack, we use the Visual Studio for Mac External Tools capability to make using the SAM CLI to publish stack to AWS easy from the IDE.

Install Steps

  1. Install/Update Visual Studio 2019 for Mac
  2. Create AWS Account - skip this step if you already have an AWS account.
  3. Install AWS CLI - skip this step if you already have the AWS CLI v2 installed.
  4. Install AWS SAM CLI - skip this step if you already have the SAM CLI installed.
  5. Install .NET CLI LazyStack Tool>

    Open a Terminal and issue this command:

    dotnet tool install lazystackapp -g
  6. Configure Visual Studio for Mac External Tools
    1. Open Visual Studio for Mac
    2. Add LazyStack -- Generate Projects tool
      1. Select Tools→Edit Custom Tools. The External Tools Pannel appears.
      2. Click the Add button. "New Tool" is added to the tools list.
      3. Select "New Tool".
      4. Edit Title to read "LazyStack - Generate Projects"
      5. Edit Command to read "/Users/yourlogin/.dotnet/lazystack". Be sure to replace yourlogin with your mac user name.
      6. Edit Arguments to read "projects"
      7. Edit Working Directory to read "${SolutionDir}"".
      8. Make sure Use output Window is checked.
    3. Add SAM Publish AWS Stack tool
      1. Click the Add button. "New Tool" is added to the tools list.
      2. Select "New Tool".
      3. Edit Title to read "SAM Publish AWS Stack"
      4. Edit Command to read "sam".
      5. Edit Arguments to read "deploy -t Stacks/Dev/serverless.template --stack-name ${SolutionName}Dev --s3-bucket samimages --capabilities CAPABILITY_NAMED_IAM"
      6. Edit Working Directory to read "${SolutionDir}"".
      7. Make sure Use output Window is checked.
    4. Add LazyStack -- Generate Stacks\Dev\AwsSettings.json tool
      1. Click the Add button. "New Tool" is added to the tools list.
      2. Select "New Tool".
      3. Edit Title to read "LazyStack - Generate Stacks\Dev\AwsSettings.json"
      4. Edit Command to read "/Users/yourlogin/.dotnet/lazystack". Be sure to replace yourlogin with your mac user name.
      5. Edit Arguments to read "settings ${SolutionName}Dev Stacks/Dev/AwsSettings.json"
      6. Edit Working Directory to read "${SolutionDir}"".
      7. Make sure Use output Window is checked.
      8. Click the OK button. The External Tools Pannel closes.
    5. Use AWS CLI to add a S3 samimages bucket
      1. Open a Terminal.
      2. Enter the following command: (change the region to match your region) 
        aws s3api create-bucket --bucket samimages --region us-east-1 --acl private

Updating CLI LazyStack Tool

To update to the latest LazyStack generator, open a Terminal and issue this command

dotnet tool update lazystackapp -g
