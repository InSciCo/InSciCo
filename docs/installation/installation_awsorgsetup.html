Part 1 - Create AWS Org

Prerequisites:
- A workstation with internet access and permissions to install software
Estimated Time: 10 minutes
Estimated Cost: none
Perform this step on the Management Workstation.
Management Workstation

This is the computer from which you will be creating/managing your AWS Organization and Accounts. You must have administrative rights on this computer to install and use a variety of software.

Names are Important

Part of setting up an effective AWS process is developing and using a consistent naming convention for the resources you are managing with that process. There is a hierarchy of AWS resources managed by the SMF that looks like this:

  • AWS Organization
    • Management Account
    • Development Organizational Unit
      • Developer Account(s)
    • Test Organizational Unit
      • System Account(s)
        • Stack(s)
    • Production Organizational Unit
      • System Account(s)
        • Stack(s)
OrgCode

Throughout the SMF process we use an organization prefix, the OrgCode, in the AWS Management Account name, Developer Account names, and AWS organizational Unit names. For instance, we refer to the AWS Management account as "OrgMgmt", but expect you to replace the "Org" with your own short OrgCode. It must be alpha-numeric and have no spaces. This short OrgCode will be very helpful in keeping track of accounts belonging to different organizations should you end up managing or developing in more than one organization on the same Workstation
Here is an example of the naming with an OrdCode of "Isc":

AWS Item Name in Docs Name with OrgCode "Isc"
Management Account OrgMgmt IscMgmt
Dev Organizational Unit OrgDevOU IscDevOU
Dev Account OrgDevJoe IscDevJoe
Test Organizational Unit OrgTestOU IscTestOU
Prod Organizational Unit OrgProdOU IscProdOU

* Note: There many be more than one development account so we append the developers "handle" to the account name; ex: IscDevJoe.

SysCode

Your organization may have one or more systems and each of these systems may contain one or more serverless stacks. You create accounts for systems under the Test Organizational Unit and Production Organizational Unit. Throughout the SMF process we use the OrgCode and a system prefix, the SysCode, in the System Account name.
Here is an example of the naming with an OrdCode of "Isc" and SysCode of "Pet":

AWS Item Name in Docs Name OrgCode "Isc", SysCode "Pet
Test Account OrgSysTest IscPetTest
Prod Account OrgSysProd IscPetProd
Stack Names in the Test Account

Test Account Stacks are published when a developer opens a GitHub Pull Request. The name of the PR branch is used to name the stack.

Production Account Stacks may be named anyway you like.

Email Account & Alias

Each Amazon account has a Root User whose credentials include an email address and password. The email address must be unique across all accounts on AWS; e.g. you may not use the same email address for more than a single AWS Account.
If you are using Gmail, you can use the Gmail dynamic aliasing technique as follows (with placeholders for OrgCode and SysCode shown):

MyEmail@gmail.com -- existing email address.

  1. MyEmail+OrgMgmt@gmail.com -- root user for management account
  2. MyEmail+OrgDevJoe@gmail.com -- root user for Joe's development account
  3. MyEmail+OrgSysTest@gmail.com -- root user for test account
  4. MyEmail+OrgProd@gmail.com -- root user for production account

The "+" character makes the email address a dynamic alias.
If you are not using Gmail, create email addresses or aliases in your email software instead.
