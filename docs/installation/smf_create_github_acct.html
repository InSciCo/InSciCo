SMF 1.1 Create GitHub Account and Configure Management Workstation

Prerequisites: none
Estimated Time: 2 - 20 minutes
Estimated Cost: none
Perform this step on the Management Workstation.

In this step we will create/configure a GitHub Account, GitHub Organization and fork the LazyStackSMF project to install the scripts and configuration files necessary to setup and run the LazyStack Software Management Framework.

GitHub Account

A GitHub account is required by LazyStack SMF. That account will have administrative rights on repositories created in your GitHub Organization that participate in the LazyStack SMF processes. LazyStack SMF requires you to configure a GitHub Organization.

GitHub Organizations

GitHub allows you create an Organization. This allows your team to collaborate across many projects while managing access to projects and data and customizing settings for your organization.

GitHub Repositories

You will be creating one GitHub repository for each serverless stack in your AWS Organization. See the User Guide GitHub Organization Integration page for more details.

GitHub CLI (gh) - GitHub Command Line Tool

We will install the official GitHub CLI tool (gh). This tool allows you to perform GitHub specific actions from the command line instead of having to log into the GitHub UI. Note: The "hub" CLI is an unofficial GitHub compatible CLI. If you are familiar with "hub", you may use that instead of "gh", but we only document the use of "gh".

Configuration Steps
  1. Visit Join GitHub to create a GitHub Account if you don't already have one.

  2. Log in to your GitHub Account and create an Organization if you don't already have one.

  3. Install "gh" as described here here.

  4. Fork the InSciCo/LazyStackSMF project
    1. Open a Terminal
    2. Issue this gh command: 
      gh auth login
      Follow the instructions to authenticate your workstation with GitHub
    3. CD to the folder under which you want to create your OrgMgmt folder. In the following steps we will fork the LazyStackSMF to create a LazyStackSMF folder and then rename that folder to OrgMgmt where you replace "Org" with your OrgCode.
    4. Issue this gh command: 
      gh repo fork InSciCo/LazyStackSMF --clone=true
      A LazyStackSMF folder will be created containing a copy of the LazyStackSMF repository.
    5. Rename the LazyStackSMF folder to "OrgMgmt". Replace "Org" with your OrgCode. The OrgMgmt folder will contain your management scripts and configuration information for the AWS Organization and accounts you will create. Access to this folder should be restricted to your Management Workstation.

  5. For the InSciCo/LazyStackSmfUtil project
    1. CD to the folder under which you created your OrgMgmt folder.
    2. Issue this gh command: 
      gh repo fork InSciCo/LazyStackSmfUtil --clone=true
      A LazyStackSmfUtil folder will be created containing a copy of the LazyStackSMF repository.

  6. Generate a Personal Access token:
    1. Log into your GitHub personal account.
    2. Click on the user profile icon in the upper right of the page. The user profile menu appears.
    3. Select the Settings menu item. The "personal account page" appears.
    4. Select "Developer Settings" from the left hand menu. The "Developer Settings" page appears.
    5. Select "Personal access tokens" from the left hand menu. The "Personal access tokens" page appears.
    6. Click on the Generate new token button. The "Confirm access" page appears. Note: GitHub may request your password at this point, provide it if requested. The "New personal access token" page appears"
    7. Enter a descriptive note, ex: "AWS CodeBuild Access".
    8. Check the "repo" option.
    9. Check the "admin:repo_hook" option.
    10. Click the "Generate token" button. The generated token is displayed.
    11. Copy this token and save it in a file named "GitHubToken.txt" in the OrgMgmt folder. Replace "Org" with your OrgCode.
    12. Sign out of the GitHub account.

  7. Install a Git client if you do not already have one installed.
    • GitBash for Windows - this is our favorite for Windows
    • Git for MacOS, Linux/Unix, Windows

  8. Install PowerShell
    1. Install PowerShell
       * If you are running Windows 10, PowerShell is already installed.

    2. Open a PowerShell terminal with "Run as Administrator" and issue the following command to change the PowerShell script execution permissions: 
      Set-ExecutionPolicy RemoteSigned
      This will allow you to run PowerShell scripts on the local computer which is generally necessary for the Management Workstation.
