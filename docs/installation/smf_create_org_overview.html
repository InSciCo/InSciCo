SMF Preparation

Prerequisites:
- A workstation with internet access and permissions to install software
Estimated Time: 10 minutes
Estimated Cost: none
Perform this step on the Management Workstation.
GitHub Organization and AWS Organization

LazyStack configures a GitHub Organization and an AWS Organization. Integrations are provided that make collaboration on the development and deployment of AWS Serverless Stacks simple. You may create a GitHub Account and Organization or configure an existing GitHub Account and Organization.

Management Workstation

This is the computer from which you will be creating/managing your GitHub Organization, AWS Organization, and Accounts. You must have administrative rights on this computer to install and use a variety of software with administrative rights.

We will configure your Management Workstation with two folders, collocated under a common folder. For example, on a Windows workstation this might look like:

  • C:\Users\User\source\repos\
    • LazyStackSMF - a GitHub clone of InSciCo/LazyStackSMF
    • LazyStackSettings - a folder containing your organization settings.

It is important to consider that the LazyStackSettings folder will contain highly sensitive information and should only be accessible by persons who administer your GitHub and AWS organizations. You may name the LazyStackSettings folder anything you like. Throughout this documentation we will refer to this folder as the LazyStackSettings folder.

OrgCode

Throughout the SMF process we use an AWS organization code, the OrgCode, in the AWS Management Account name, Developer Account names, and AWS organizational Unit names. For instance, we refer to the AWS Management account as "OrgMgmt", but expect you to replace the "Org" with your own short OrgCode. It must be alpha-numeric and have no spaces. This short OrgCode will be very helpful in keeping track of accounts belonging to different organizations should you end up managing or developing in more than one organization on the same Workstation.
Here is an example of the naming with an OrdCode of "Isc":

AWS Item Name in Docs Name with OrgCode "Az"
Management Account OrgMgmt AzMgmt
Dev Organizational Unit OrgDevOU AzDevOU
Dev Account OrgDevJoe AzDevJoe
Test Organizational Unit OrgTestOU AzTestOU
Prod Organizational Unit OrgProdOU AzProdOU

* Note: There many be more than one development account so we append the developers "handle" to the account name; ex: AzDevJoe.

Your GitHub Organization can be named anyway you like. You may use an existing GitHub Organization instead of creating one from scratch.

SysCode

Your AWS organization may have one or more systems and each of these systems may contain one or more serverless stacks. Each System has one System Test AWS Account and one System Production AWS Account. Throughout the SMF process we use the OrgCode and a system code, the SysCode, in the System Account name.
Here is an example of the naming with an OrdCode of "Az" and SysCode of "Pet":

AWS Item Name in Docs Name OrgCode "Az", SysCode "Pet"
Test Account OrgSysTest AzPetTest
Prod Account OrgSysProd AzPetProd
Stack Names in the Test Account

Test Account Stacks are published when a developer opens a GitHub Pull Request. The name of the PR branch is used to name the stack.

Production Account Stacks may be named anyway you like as long as they follow AWS Stack Name conventions.

Email Account & Alias

Each Amazon account has a Root User whose credentials include an email address and password. The email address must be unique across all accounts on AWS; e.g. you may not use the same email address for more than a single AWS Account.
If you are using Gmail, you can use the Gmail dynamic aliasing technique as follows (with placeholders for OrgCode and SysCode shown):

MyEmail@gmail.com -- existing email address with an OrgCode of "Az" and SysCode of "Pet".

  1. MyEmail+AzMgmt@gmail.com -- root user for management account
  2. MyEmail+AzDevJoe@gmail.com -- root user for Joe's development account
  3. MyEmail+AzPetTest@gmail.com -- root user for system test account
  4. MyEmail+AzPetProd@gmail.com -- root user for system production account

The "+" character makes the email address a dynamic alias.
If you are not using Gmail, create email addresses or aliases in your email software instead.
